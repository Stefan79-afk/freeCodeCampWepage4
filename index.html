Classes

A simple way to think about classes is that they are like blueprints in the real world: In a blueprint, you clearly draw the schematics for a particular building you want to build, by defining its shape, its size and so on. In much the same way, a class is a blueprint for objects: you declare an object's variables, its methods(functions that they can call)

Objects

If classes are the blueprint, then objects are the actual result of that blueprint.An object, also called instance of the class, is a data type composed of the class's variables that can use its methods.

#include "iostream"

using namespace std;

class Complex
{
    int real;
    int imaginary;

    void display_nr()
    {
        cout << real << " + i *" << imaginary << endl;
    }
};

int main()
{
    Complex a;

    a.real = 5;
    a.imaginary = 7;
    a.display_nr();

    return 0;

}

The code above demonstrates the basics of classes and objects: the class Complex defines the blueprint/definition of a complex number, made from a real part and an imaginary part. The method/function display_nr simply outputs to the console the 2 variables in the way a complex number is written. Then the object a gets declared, and its variables are given values. The object can call upon the method and have its values written in the usual form.

What are they?

Access types refer to where the classe's contents can be used. A simple way to think about it is like asking for permission. Different types of access can or cannot offer permision for outside code to modify the data within the class. The 3 different types are:

  • Private
  • Public
  • Protected
They will all be expanded upon in the following paragraphs

Private

By default, the variables of a class are declared as private, which means that they cannot be modified outside of the classe's scope. Similarly, if a function is declared in private, it cannot be called outside the class. It is often good practice to declare the variables of a class as private, in order to avoid illegal access and modifications.

#include "iostream"

using namespace std;

class Complex
{
    private:
        int real;
        int imaginary;

    void display_nr()
    {
        cout << real << " + i *" << imaginary << endl;
     }
};

int main()
{
    Complex a;

    a.real = 5; //This will cause an error, since real is not accessible outside of "Complex"
    a.imaginary = 7;
    a.display_nr();

    return 0;

}

Public

Elements declared as public can be accessed from anywhere in the program, therefore making this type the most permissive. A class's methods are generally declared as public

Protected

A protected element can be accessed by the class and classes that inherit it. Inheritance will be disscussed more in-depth in the Inheritance section

Constructors

A constructor is a special function that has no type and does not return anything. It is used to initialize the variables of an object. For example, in the code snippet above, instead of declaring each of a, we can do this instead:

#include "iostream"

using namespace std;

class Complex
{
    Complex(int a, int b) //parameters
    {
        this->real = a;
        this->imaginary = b;
    }//this is the constructor
};
.
.
.
a(5, 7);

It is also the only way to declare the variables if they are declared as private.

Deconstructors

In C++, you can manually allocate memory for variables or data structures, as well as free that memory. If variables are not manually allocated, they are deleted when the program exits their scope / when the program terminates. Therefore, objects follow the same principle. They are deleted when their scope ends / the program terminated. Objects, like any other variable, can also be manually allocated in the heap, and can also be freed. The latter part is where the deconstructor comes in. Like a Constructor, it is a special function that has no type and no return. It is simply used to free the memory of all the variables in the objects:

#include "iostream"

using namespace std;

class Example
{
    private:
        int *a = new int;
    int *b = new int;


    Example(int a, int b)
    {
        *this->a = a;
        *this->b = b;
    }//Constructor

    ~Example()//Declared as class name preceded by ~
    {
        delete a;
        delete b;
    }
};

What is Inheritance?

Like in the real world, a class can inherit the attributes of another class. In Inheritance, the class that inherits is often called the derived class, while the class that is being inherited from is known as the base class.

#include "iostream"

using namespace std;

class A
{
    //variables and methods
};

class B:public A
{
    //besides it's own attirbutes, B can also use A's, as long as they are not private.
};

In this example, class B inherits A's attirbutes publically. By publically, it means that it basically inherits everything that is not private (more on this later). Now, B can use A's functions and variables, if the respective variables/functions are public/protected.

Protected Access

As mentioned before, access type protected is associated with the concept of Inheritance. If a variable in class A is declared as protected, then all classes that derive A(like B) have permission to modify the variable's value. Besides the derived classes, the variable is untouchable elsewhere in the program

Inheriting with public vs protected vs private

As you've seen in the code snippet above, class B inherited public A. To explain this easily, it's best to just showcase this with a code snippet:

#include "iostream"

using namespace std;

class A
{
    public:
        int x;
    protected:
        int y;
    private:
        int z;
};

class B : public A
{
    // x is public
    // y is protected
    // z is not accessible from B
};

class C : protected A
{
    // x is protected
    // y is protected
    // z is not accessible from C
};

class D : private A // 'private' is default for classes
{
    // x is private
    // y is private
    // z is not accessible from D
};

What is Polymorphism?

The word Polymorphism is composed of the words Poly, which means many and morphism, which means form. Therefore, polymorphism translates roughly to multiform. This defenition makes sense, given the way it works in OOP.
In OOP, polymorphism is generally thought of as a function that is shared between a base class and its derived classes with the same name that executes differently depending on the type(class) of the object. It basically changes its "form" depending on its class.

#include "iostream"

using namespace std;

class A
{
    void display()
    {
        cout << "I am class A" << endl;
    }

};


class B:public A
{
    void display()
    {
        cout << "I am class B" << endl;
    }

};


class C:public A
{
    void display()
    {
        cout << "I am class C" << endl;
    }

};

In the exmaple above, A is the base class, while B and C derive from A. All 3 classes have the same method display(), but print something different depending on their class

Virtual Functions

A virtual function is basically a template function used in polymorphism. The base class's function is declared as virtual, which makes it unable to be executed unless using pointers. Then the derived classes use the same function(minus the virtaul keyword). This is useful when an object of a derived class is accessed using a pointer of a base class: if the base class's function is not virtual, the pointer would call it. But if it's virtual, the pointer will instead call the function of the derived class's object

#include "iostream"

using namespace std;

class A
{
    virtual void display()
    {
        cout << "I am base class" << endl;
    }

};


class B:public A
{
    void display()
    {
        cout << "I am class B" << endl;
    }

};


int main()
{
    A* a;
    B b;
    a = &b
    a.display()//will use display of B, not A
}

Static

A static function / variable is basically a function/variable that is shared between classes. If 4 classes were to be declared with a static variable, that variable would have the same value for all 4 objects of the declared classes

#include "iostream"

using namespace std;

class A
{
    static a = 18;

    void display_value()
    {
        cout << this->a; << endl;
    }
};


int main()
{
    A a, b, c, d;
    a.display();b.display;c.display();d.display();//they will all print 18
}

Copy Constructor

A copy constructor is called when an object is assigned the value of another object. It basically handles the reassingment of the variables of the new object to the values of the assigned objects. Like a regular constructor, is has the name of the class and doesn't return anything.

#include "iostream"

using namespace std;

class A
{
    int a, b;

    A(int a, int b)
    {
        this->a = a;
        this->b = b;
    }

    A(const A &other)//A constant reference to the object that will be used to assign its values to the current one
    {
        this->a = other.a;
        this->b = other.b;
    }
};


int main()
{
    A a(2, 3);
    B b = a//this will call the copy construtor, and give b.a value 2 and b.b value 3
}

Operator Overloading

Operator overloading, like function overloading, involves alterating the behavior of an operator(think +, -, and so on) under different circumstances. It is often used in OOP to define operations between objects. The simplest and most popular example is with string concatination: strings are objects, and so they overload the + to stick the 2 strings together

#include "iostream"

using namespace std;

class Complex
{
    int real, imaginary;

    Complex(int a, int b)
    {
        this->real = a;
        this-imaginary = b;
    }

    Complex operator+ (const A &second)//second is the second operand of +, first is this
    {
        Complex result;//the third number that is the result of the operation
        result.real = this->real + second.real;
        result.imaginary = this->imaginary + second.imaginary;
        return result
    }
};


int main()
{
    Complex A(2, 3);
    Complex B(4, 5);
    Complex C = A + B//C.real = 6, C.imaginary = 8
}
